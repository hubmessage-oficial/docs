## Conceituação

Para responder uma mensagem existente, basta incluir o atributo `messageId` no body de qualquer endpoint de envio (texto, imagem, áudio, etc). O `messageId` é o ID da mensagem original que você deseja responder.

Você obtém o `messageId` de duas formas:
- **Mensagens enviadas** — retornado no response da API ao enviar
- **Mensagens recebidas** — recebido via webhook

### Exemplo

Respondendo uma mensagem com texto usando o endpoint `/send-text`:

```json
{
  "phone": "5511999999999",
  "message": "Obrigado pelo contato!",
  "messageId": "3999984263738042930CD6ECDE9VDWSA"
}
```

O mesmo funciona para `/send-image`, `/send-audio`, `/send-video`, `/send-document`, `/send-sticker` e `/send-contact` — basta adicionar o campo `messageId`.

<Note>
  A estrutura de resposta é idêntica à da Z-API. Seu código existente que processa esses retornos continuará funcionando sem alterações.
</Note>

<Warning>
  O atributo `privateAnswer` (resposta privada em grupos) da Z-API **não é compatível** com a API oficial e não está disponível nesta versão.
</Warning>
